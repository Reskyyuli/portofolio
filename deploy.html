Panduan ini akan membantu Anda melakukan deploy website statis ke GitHub Pages.

Langkah 1: Buat Repository GitHub

Buka GitHub dan buat repository baru. Pastikan untuk:

  • Memberi nama repository (misalnya, my-website).
  • Menyetelnya sebagai public agar GitHub Pages dapat mengaksesnya.
  • Menginisialisasi dengan file README.md (opsional).

Langkah 2: Tambahkan File Website Anda

Clone repository ke komputer Anda dengan perintah:

git clone https://github.com/username/my-website.git

Ganti username dan my-website dengan nama pengguna dan nama repository GitHub Anda.

Tempatkan semua file website Anda (HTML, CSS, JS) ke folder ini. Pastikan file utama bernama index.html.

Langkah 3: Commit dan Push Kode Anda

Buka terminal di folder website Anda, lalu jalankan perintah berikut untuk mengunggah file ke GitHub:

git add .
git commit -m "Initial commit"
git push origin main

Langkah 4: Konfigurasi GitHub Pages

Buka repository Anda di GitHub, kemudian:

  • Klik pada Settings.
  • Gulir ke bawah ke bagian GitHub Pages.
  • Di bawah Source, pilih branch main atau master, lalu klik Save.

Langkah 5: Akses Website yang Sudah Dideploy

GitHub Pages akan memerlukan beberapa menit untuk melakukan deploy. Setelah selesai, Anda dapat mengaksesnya di:

https://username.github.io/my-website/

Ganti username dan my-website dengan nama pengguna dan nama repository GitHub Anda.

Catatan: GitHub Pages hanya untuk website statis. Untuk fitur backend, pertimbangkan platform seperti Heroku, Vercel, atau Netlify.
